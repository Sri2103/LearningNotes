# DevOps / SRE Learning Framework

## 1. Why am I learning DevOps/SRE?

> To build scalable, automated, reliable systems, reduce toil, ensure uptime, and enable fast, safe software delivery.

## 2. Subskills (Decomposition)

- Linux & Networking
- Version control & GitOps
- CI/CD pipelines
- Infrastructure as Code
- Cloud fundamentals
- Kubernetes operations
- Monitoring & observability
- Incidents & on-call
- SRE principles
- Reliability engineering
- Automation & tooling
- Security basics

## 3. Knowledge Map

Fundamentals → Linux, networking, Git, CI/CD  
Intermediate → IaC, Docker, Kubernetes, monitoring  
Advanced → SRE practices, HA systems, autoscaling, infra automation

## 4. Core Resources

Books:

- Google SRE Book
- SRE Workbook
- The DevOps Handbook
- Accelerate

Courses:

- Kubernetes (CKA)
- Terraform + Ansible tracks
- Observability (Prometheus/Grafana)

## 5. Learning Method

Learn → build → automate → test → break → fix → improve

## 6. Practice Plan

Daily: 30 min automation  
Weekly: infra lab  
Monthly: a full mini SRE scenario

## 7. DevOps/SRE Playbook

1. Identify the problem area (infra/app/obs/network)
2. Automate repetitive steps
3. Observe system behavior
4. Implement reliability patterns
5. Measure SLIs/SLOs
6. Reduce toil
7. Write runbooks
8. Validate post-incident

## 8. Common Mistakes

- Manual deployments
- No Infrastructure-as-Code
- No monitoring/alerting
- Patching in prod manually
- Ignoring logs
- Overcomplicating automation
