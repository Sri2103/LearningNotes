# Linux Learning Framework

## 1. Why am I learning Linux?

> To become confident in servers, CLI, debugging, performance tuning, and production systems.

## 2. Subskills (Decomposition)

- Shell & CLI
- Filesystems & permissions
- Processes & signals
- Packages & services
- Networking
- Logs & monitoring
- System internals
- Performance tuning
- Kernel basics

## 3. Knowledge Map

Fundamentals → CLI, processes, permissions  
Intermediate → networking, logs, services  
Advanced → cgroups, namespaces, kernel tracing

## 4. Core Resources

Books:

- How Linux Works
- Linux Command Line
- Linux Performance (Brendan Gregg)

Courses:

- Linux Foundation
- DigitalOcean Linux tutorials

## 5. Learning Method

Command → experiment → break → fix → document

## 6. Practice Plan

Daily: run 3 commands deeply  
Weekly: debug 1 real system issue  
Monthly: complete a Linux project

## 7. Linux Playbook

1. Identify subsystem (CPU/mem/io/network/fs)
2. Use appropriate commands
3. Analyze logs
4. Trace system calls
5. Drill down using perf tools
6. Document findings

## 8. Common Mistakes

- Running commands blindly
- Misunderstanding permissions
- Using sudo everywhere
- Not checking logs
