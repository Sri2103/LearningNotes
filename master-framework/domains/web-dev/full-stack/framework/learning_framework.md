# Web Development (Frontend + Backend) Learning Framework

## 1. Why am I learning this?

> To build full-stack applications, understand the web deeply, design scalable frontends/backends, and deploy production-grade systems.

## 2. Subskills (Decomposition)

### Frontend

- HTML, CSS
- Flexbox, Grid
- JavaScript ES6+
- TypeScript
- React (or Vue/Svelte)
- State management
- Client-side routing
- UI/UX basics

### Backend

- HTTP fundamentals
- REST APIs
- authentication & sessions
- databases (SQL/NoSQL)
- ORMs
- caching
- file handling
- queues
- microservices

### Full-stack Engineering

- API design
- security (OWASP)
- deployments
- docker
- CI/CD
- performance

## 3. Knowledge Map

Fundamentals → HTML/CSS/JS & HTTP  
Intermediate → React, REST APIs, auth  
Advanced → performance, scaling, microservices, deployment

## 4. Core Resources

Books:

- You Don’t Know JS
- Eloquent JavaScript
- Web Security for Developers
- Designing Web APIs

Courses:

- Full-stack open
- Modern React
- Node.js specialization
- CS50 Web

## 5. Learning Method

Build → break → debug → improve → scale

## 6. Practice Plan

Daily: 30 min coding  
Weekly: 1 small component/API  
Monthly: 1 real project

## 7. Web Dev Playbook

1. Start with wireframes
2. Define API contract
3. Implement backend first
4. Build frontend components
5. Add authentication
6. Strengthen security
7. Add caching & performance
8. Deploy & monitor

## 8. Common Mistakes

- no responsive design
- forgetting async/await
- using any() in TypeScript
- SQL injection risks
- oversized API payloads
- no validation
